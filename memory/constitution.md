# Influenter 專案憲章

## 專案願景

Influenter 致力於成為台灣創作者的最佳合作案件管理夥伴，透過 AI 技術將繁瑣的溝通與管理工作自動化，讓創作者能專注於內容創作本身。

## 核心價值觀

### 1. 使用者至上
- 以台灣創作者的實際工作流程為設計核心
- 介面直覺易用，降低學習成本
- 尊重使用者隱私與資料安全

### 2. AI 賦能而非取代
- AI 作為輔助工具，最終決策權在使用者手上
- 提供 AI 信心指標，讓使用者了解建議的可靠度
- 允許使用者修正 AI 判斷，持續改善準確度

### 3. 漸進式價值交付
- 優先實作核心功能（Gmail 整合、案件管理）
- 快速迭代，根據回饋調整
- 保持系統可擴展性

### 4. 技術卓越
- 編寫乾淨、可維護的程式碼
- 完善的錯誤處理與日誌記錄
- 重視安全性與效能

## 技術決策原則

### 架構設計
- **前後端分離**：清晰的 API 邊界
- **模組化**：每個功能模組可獨立開發與測試
- **API First**：先定義 API 規格再實作
- **資料驅動**：所有狀態變更可追蹤

### 程式碼品質
- 遵循 Go 與 Vue 的最佳實踐
- 單元測試覆蓋率 > 70%
- 使用 TypeScript 提升前端程式碼品質
- 統一的程式碼風格（gofmt、ESLint/Prettier）

### 安全性
- 敏感資料加密儲存（Gmail tokens、API keys）
- 實作 CSRF 保護與 XSS 防護
- 定期進行依賴套件安全性檢查
- 遵循 OWASP Top 10 安全指南

### 效能
- API 回應時間 < 200ms (P95)
- 前端首次載入 < 3 秒
- 支援分頁與無限捲動
- 資料庫查詢優化（適當的索引）

## 不做什麼

1. **不做社群媒體排程**：專注於案件管理，不與現有排程工具競爭
2. **不做金流處理**：初期不處理實際付款，僅記錄報價與收款狀態
3. **不做多語言版本**：初期專注台灣市場（繁體中文）
4. **不過度設計**：避免為未來可能的需求增加複雜度

## 使用者隱私承諾

- 郵件資料僅用於系統功能，不做其他用途
- 不販售或分享使用者資料給第三方
- 提供資料匯出與刪除功能
- 明確的隱私政策與使用條款

## 成功指標

### MVP 階段（前 3 個月）
- 成功連接 Gmail 並同步郵件
- AI 正確分類郵件準確率 > 80%
- 能自動抽取基本資訊（品牌、日期）
- 完成至少 50 位創作者的 beta 測試

### 成長階段（6-12 個月）
- 月活躍使用者 > 500
- 使用者每週平均處理 > 5 個案件
- AI 生成回覆的採用率 > 60%
- 使用者滿意度 NPS > 40

## 開發原則

### 版本控制
- 使用語義化版本（Semantic Versioning）
- 清晰的 commit 訊息（Conventional Commits）
- 功能分支開發，PR review 後合併

### 部署策略
- 開發、測試、正式環境分離
- 自動化 CI/CD 流程
- 資料庫遷移腳本版本控制
- 零停機部署

### 文件化
- API 文件自動生成（Swagger/OpenAPI）
- 重要決策記錄在 ADR（Architecture Decision Records）
- 使用者文件與操作手冊
- 開發者指南與設置說明

## 團隊協作

- 每週同步進度與遇到的問題
- 開放式溝通，鼓勵提問與分享
- 程式碼審查是學習機會，保持友善
- 慶祝小勝利，從失敗中學習
